# OA 办公管理系统 - 接口自动化测试计划

## 1. 项目概述

- **项目名称：** OA 办公管理系统
- **测试目标：** 建立可重复执行的接口自动化测试，覆盖基础功能、权限、异常、跨服务闭环和主要扩展能力
- **测试负责人：** 王良凤
- **测试类型：** 黑盒接口测试（基于 Gateway 统一入口）
- **技术栈：** Java 21, JUnit 5, JDK HttpClient, Jackson, Maven
- **契约工具：** Apifox（OpenAPI 3.1 同步与管理）
- **测试日期：** 2026-07-22

## 2. 测试范围

### P0（必须完成）

| 模块 | 接口 | 核心验证点 |
|------|------|-----------|
| 认证 | POST /auth/login, POST /auth/logout | 成功登录/退出、错误密码、禁用账号、Token 校验 |
| 组织与 RBAC | 用户/部门/岗位 CRUD | 正常增删改查、关联约束、角色权限边界 |
| 考勤 | 打卡/状态/记录 | 上下班打卡、重复打卡、数据隔离 |
| 审批 | 申请/待办/审批 | 请假加班提交、审批流、自审拒绝 |
| 公告 | 发布/列表/已读 | 管理员发布、员工查看、未读数变化 |
| 业务闭环 | 跨服务场景 | 打卡->查询、申请->审批->消息、公告->已读 |

### P1（时间允许时完成）

| 模块 | 验证点 |
|------|--------|
| XXL-Job 核算 | 打卡核算幂等执行 |
| Excel 导出 | 文件可打开、数据一致 |
| Flowable 多级审批 | 两级请假流程 |
| RocketMQ 通知 | 审批事件消息幂等 |
| Elasticsearch 检索 | 中文搜索、索引重建 |
| AI 接口 | 超时、限流、降级 |
| Sentinel 限流 | 限流熔断恢复 |

## 3. 测试策略

- 所有测试通过 Gateway（http://localhost:8088）执行
- 每个测试方法独立准备数据，不依赖执行顺序
- 登录后自动从响应提取 Token，注入后续请求
- 使用 test.properties 管理非敏感配置
- 敏感信息（密码、Token）不在日志和报告中完整输出
- 外部中间件未启动时标记为阻塞，不修改测试为假通过
